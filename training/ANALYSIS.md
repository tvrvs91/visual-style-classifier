# Анализ: что не так с моделью и что чинить

Документ собран по результатам двух тренировок (EfficientNet-B0 и ResNet-50)
и по поведению на пользовательских кадрах. Цель — определить **где реальное
узкое место**, а не просто наклепать «улучшений».

---

## TL;DR

| Что чинить | Ожидаемый прирост val acc | Стоимость по времени |
|---|---:|---:|
| 1. Улучшить датасет (баланс + чистка) | **+5…12 пп** | высокая (часы разметки) |
| 2. Сократить классы (8 → 5–6) | **+5…10 пп** | средняя (1–2 часа) |
| 3. Class balancing + двухфазный fine-tune | +3…5 пп | низкая (готово в `train.py`) |
| 4. Перейти на ResNet-50 | +3…5 пп | низкая |
| 5. Test-time augmentation на инференсе | +1…2 пп | низкая |
| 6. EfficientNet-B2 / B3 | +1…3 пп | низкая |

**Рекомендация:** идти строго по порядку. Архитектура и трюки обучения дают
крошки относительно качества данных.

---

## Что мы знаем

### Метрики обучения

| Модель | Best val acc | Эпоха | Train loss → val loss | Состояние |
|---|---:|---:|---|---|
| EfficientNet-B0 | 0.678 | 5/15 | 0.86 → 0.95 | Плато на 5-й эпохе |
| ResNet-50 | 0.728 | 8/10 | 0.64 → 0.78 | Не вышел на плато |

### Поведение на тестовых кадрах

5 фотографий (полароид Шварца, кухня в золотом свете, мим, сад, Сол Гудман):

- 2/5 уверенных хитов
- 1/5 частичный
- **1/5 явный мисс** — кухня в золотом свете → `street + minimalist` (должно быть `golden_hour`)
- **4/5 содержат `moody`** в топ-2 — сильный bias на «дефолтный» класс

Раньше (на другой модели) был bias на `minimalist`. Сейчас сместился на `moody`.
**Это паттерн** — модель находит «безопасный» класс и ставит его всегда.

## Корневые причины (по убыванию вероятности)

### 1. Несбалансированный или зашумлённый датасет (вероятность 70%)

**Диагноз.** Bias на доминирующий класс — характерный симптом перевеса
в данных. Если в `moody/` сейчас в 2–3 раза больше фото, чем в `golden_hour/`,
модель учится отвечать `moody` как страховку — это снижает ожидаемый loss
на дисбалансной выборке.

**Доказательства.**
- Plateau на 5-й эпохе у EfficientNet — типично когда модель «всё что могла из этого
  объёма данных, выучила». Capacity B0 (5.3M параметров) больше, чем нужно для
  такого датасета.
- Gap train↔val 0.09 — overfitting уже начался, значит данных мало.
- Class drift между тренировками (minimalist → moody) — модель находит разные
  spurious correlations при разных random seed'ах. Признак шумных меток.

**Что делать.**
- Запустить `dataset_check.py` (см. ниже) на трейне — посмотреть распределение
  классов, дубликаты, повреждённые файлы.
- Цель: ≤ 1.5x разница между классами, минимум 200 фото на класс.
- Перепросмотреть глазами 30 случайных фото на класс — выкинуть пограничные.

### 2. Размытые границы классов (вероятность 20%)

**Диагноз.** Классы `moody`, `dark`, `vintage`, `dramatic` визуально пересекаются.
Один и тот же кадр может честно принадлежать сразу 2–3 классам. Single-label
классификация в такой ситуации заставляет модель «выбирать» — и она выбирает
тот класс, что чаще встречался в трейне.

**Что делать (любой из вариантов).**
- **(a) Сократить классы.** Выкинуть `moody` (он перекрывается с `dark` и
  `dramatic`); опционально объединить `dark + dramatic` → `cinematic`. Останется
  5–6 чётких классов: `airy, golden_hour, minimalist, street, vintage,
  cinematic`. Это самое рентабельное по соотношению эффект/затраты.
- **(b) Переразметить.** Жёстко: каждое фото — ровно один однозначный класс.
  Пограничные — выкинуть.
- **(c) Multi-label.** Заменить softmax на sigmoid + BCE loss; разрешить
  модели выдавать несколько классов одновременно. Точнее отражает реальность,
  но требует переразметки в multi-label формате.

### 3. Архитектура (вероятность 5%)

ResNet-50 показал +5 пп при идентичных данных — значит EfficientNet-B0 был
капасити-ограничен на этом датасете. Но ResNet-50 ещё не вышел на плато,
а датасет тот же — значит **дата всё ещё узкое место**, просто более
сильная модель её лучше выжимает.

**Что делать.** Имеет смысл, но **только после** фиксов 1 и 2. Иначе получим
ту же модель, ту же ошибку, но ×6 тяжелее.

### 4. Гиперпараметры обучения (вероятность 5%)

Базовый fine-tuning без двухфазности, без cosine schedule, без label smoothing,
без class balancing — это «по букварю», но не оптимум. Уже всё реализовано
в `train.py`.

## План действий — по приоритетам

### Шаг 0. Диагностика (15 минут)

```bash
# на машине где есть датасет
python training/dataset_check.py --data-dir /path/to/dataset

# на текущей обученной модели — узнаем какие классы путаются
python training/diagnose.py --weights /path/to/efficientnet_b0_styles_best.pth \
                            --val-dir /path/to/dataset/val
```

Это даст:
- распределение по классам
- топ-10 путаниц (моделью говорит X, на самом деле Y)
- per-class precision / recall / F1
- честный confusion matrix

**После этого станет ясно:** проблема в данных или в модели. Без этого шага
дальнейшие действия — гадание.

### Шаг 1. Если есть дисбаланс или шум (наиболее вероятно)

1. Дополнить недопредставленные классы — Unsplash / Pexels / собственный
   архив. Цель: каждый класс ≥ 300 фото в трейне.
2. Прокликать визуально по `moody/` — оставить только то, что НЕ влезает в
   `dark`, `vintage`, `dramatic`. Скорее всего останется мало — это сигнал
   что класс мусорный.
3. Если решено выкинуть `moody`:
   ```bash
   # удалить папку moody/ из dataset/train и dataset/val
   # обновить ml-service/app/config.py — убрать "moody" из styles
   # написать миграцию backend V2__remove_moody.sql
   ```

### Шаг 2. Перетренировать с улучшенным pipeline'ом

```bash
python training/train.py \
    --data-dir /path/to/dataset \
    --arch efficientnet_b0 \
    --epochs 30 \
    --head-epochs 5
```

Что здесь уже делается из коробки:
- WeightedRandomSampler (балансирует классы внутри минибатча)
- Двухфазный fine-tune: 5 эпох голова, 25 — всё с cosine LR
- Label smoothing 0.1
- Сильная аугментация (но БЕЗ grayscale и сильного hue jitter — сохраняет
  warmth-сигнал нужный для golden_hour и vintage)
- Mixed precision на GPU

Ожидаем: с теми же данными, но с этим pipeline'ом → +3–5 пп к val acc.
В сумме с фиксами шага 1: **0.78–0.85 val acc реально достижимо**.

### Шаг 3. Если хочется ещё (после шагов 1–2)

- Перейти на `--arch resnet50` — стабильно даёт +3–5 пп, но размер модели
  ×6 и инференс ×3 медленнее
- Или попробовать EfficientNet-B2 (промежуточный размер, в 2 раза больше B0)
- Включить TTA в ml-service (`USE_TTA=true`) — +1–2 пп бесплатно на инференсе

### Шаг 4. Контрольные тесты после переобучения

Прогнать на тех же 5 фотографиях, что вылезали раньше:
- Кухня в золотом свете → должна стать `golden_hour` (топ-1)
- Полароид Шварца → должна стать `vintage` (топ-1)
- Сол Гудман → должна остаться `dramatic + moody`/`dark`
- Сад → может быть любым низкосаюрным классом

Если 4 из 5 правильно — модель в реально хорошей форме для дипломной защиты.
