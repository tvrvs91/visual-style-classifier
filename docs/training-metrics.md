## Анализ

### EfficientNet-B0
- **Плато с эпохи 5.** Train loss продолжает падать (0.97 → 0.86), val loss/acc стоят. Лёгкое переобучение, но не катастрофическое.
- **Финальный gap train↔val:** train ≈ 0.86, val ≈ 0.95 → разница ~0.09. Терпимо.
- **Аргумент в пользу:** в 5–6 раз меньше и быстрее. Хорош для production / mobile.

### ResNet-50
- **Кривая ещё не вышла на плато к 10 эпохе** — train и val loss продолжают снижаться. Можно было бы выжать ещё 1–3 пп точности продолжив обучение.
- **Лучше генерализуется**: val loss 0.78 против 0.95 — модель более уверена в правильных предсказаниях, не только в топ-1.
- **Аргумент против:** в 6 раз тяжелее по диску, ×3 по времени инференса.

### Вывод для диплома
Обе модели — рабочий результат для 8-классовой задачи на ограниченном датасете
(67–73% top-1 — стандартный диапазон для такого размера и сложности). ResNet-50
объективно точнее на ~5 процентных пунктов, но EfficientNet-B0 даёт лучший
баланс точность/инфраструктура — оправданный выбор для production-сервиса
с CPU-инференсом.

В отчёте можно представить **обе модели как сравнительный эксперимент**,
показывающий обоснование выбора EfficientNet-B0 для финального деплоя.

## Что бы я улучшил для следующей итерации

1. **Дольше обучать ResNet-50** — кривая не насыщена, ещё 5–10 эпох вероятно дадут 73–75% val acc.
2. **Per-class confusion matrix** на validation set — для диагностики, какие классы путаются. Сейчас по поведению на пользовательских кадрах видно сильный bias класса `moody`.
3. **Class balancing** — `WeightedRandomSampler` или `class_weight` в loss'е, если посчитать что распределение по классам неравномерное.
4. **Test-time augmentation** — усреднение по 5 аугментациям одного кадра — стабильно даёт +1–2 пп на классификации.

## Готовность к графикам диплома

Для построения двух кривых (по эксперименту):

```
X: epoch
Y1: train_loss (синий)
Y1: val_loss (оранжевый)
Y2 (правая ось): val_accuracy (зелёный)
```

Данные выше уже структурированы — копировать в pandas / Excel / matplotlib
напрямую без переразметки.
